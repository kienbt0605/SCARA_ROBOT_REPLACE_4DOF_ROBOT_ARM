**THE 4DOF ROBOT ARM PROJECT HAS BEEN REPLACED WITH SCARA_ROBOT** 
🤖 SCARA ROBOT FIRMWARE
1. 📖 PROJECT OVERVIEW
THIS PROJECT PROVIDES FIRMWARE FOR A 3-DOF (2R + Z) SCARA ROBOT BUILT ON AN ARDUINO UNO AND A CNC SHIELD V3. IT ENABLES PRECISE CONTROL OVER JOINT ANGLES (FORWARD KINEMATICS) AND CARTESIAN COORDINATES (INVERSE KINEMATICS). THE SYSTEM SUPPORTS LINEAR AND CIRCULAR INTERPOLATION FOR DRAWING TASKS AND COMMUNICATES WITH A PYTHON GUI VIA CONTINUOUS SERIAL COMMANDS.

2. ⚙️ HARDWARE ARCHITECTURE
    CONTROLLER: ARDUINO UNO WITH CNC SHIELD V3

    MOTORS: STEPPER MOTORS FOR JOINT 1 (SHOULDER), JOINT 2 (ELBOW), AND Z-AXIS

    SENSORS: LIMIT SWITCHES FOR HOMING JOINT 1 AND JOINT 2

    LINK LENGTHS: L1 (UPPER ARM) = 228.0 MM, L2 (FOREARM) = 136.5 MM

3. ✨ KEY FEATURES
    MANDATORY HOMING SYSTEM: CONCURRENT HOMING SEQUENCE USING LIMIT SWITCHES TO ESTABLISH A RELIABLE (0,0) ORIGIN. THE ROBOT WILL REJECT COMMANDS UNTIL HOMING IS COMPLETED.

    FORWARD & INVERSE KINEMATICS: AUTOMATIC CALCULATION OF END-EFFECTOR POSITIONS FROM ANGLES, OR REQUIRED ANGLES FROM XY COORDINATES. IT EXCLUSIVELY UTILIZES AN ELBOW-UP CONFIGURATION.

   WORKSPACE VALIDATION: BUILT-IN MATHEMATICAL CHECKS TO PREVENT THE ROBOT FROM ATTEMPTING TO MOVE OUTSIDE ITS PHYSICAL REACH.

   SMOOTH DRAWING MODES: DYNAMIC SPEED AND ACCELERATION ADJUSTMENTS (HIGH ACCELERATION, LOWER SPEED) TO ENSURE SMOOTH LINE AND CIRCLE GENERATION USING MICRO-STEPPING INTERPOLATION.

4. 🔌 SERIAL COMMUNICATION PROTOCOL
THE FIRMWARE LISTENS FOR SERIAL COMMANDS AT 115200 BAUD AND ROUTES THEM BASED ON THE INITIAL CHARACTER:

    H : HOMING - FIND THE ORIGIN VIA LIMIT SWITCHES.

    J,<JOINT>,<DELTA> : JOG - RELATIVE MOVEMENT OF A SPECIFIC JOINT.

    A,<Θ1>,<Θ2>,<Z> : ANGLE - MOVE TO ABSOLUTE JOINT ANGLES.

    P,<X>,<Y> : POINT - MOVE TO ABSOLUTE XY CARTESIAN COORDINATES USING INVERSE KINEMATICS.

    D,<X1>,<Y1>,<X2>,<Y2>,<Z_UP>,<Z_DOWN> : DRAW LINE - EXECUTES PEN UP, RAPID TRAVEL TO START, PEN DOWN, AND LINEAR INTERPOLATION.

    C,<CX>,<CY>,<R>,<Z_UP>,<Z_DOWN> : DRAW CIRCLE - EXECUTES PEN UP, RAPID TRAVEL TO EDGE, PEN DOWN, AND CIRCULAR INTERPOLATION.

AFTER EVERY COMMAND EXECUTION (OR ERROR), THE ARDUINO REPORTS ITS CURRENT STATE BACK TO THE HOST:
📡 STATUS,<X>,<Y>,<Z>,<Θ1>,<Θ2>

5. 🏗️ SOFTWARE LAYERS
 COMMUNICATION LAYER: CONTINUOUSLY PARSES INCOMING SERIAL COMMANDS AND REPORTS THE ROBOT'S KINEMATIC STATUS TO THE GUI.

 CONTROL LAYER: MANAGES LIMIT SWITCH HOMING, MOTOR ENABLE/DISABLE STATES, AND CALLS THE KINEMATIC SOLVERS FOR TARGET CALCULATIONS.

 DRAWING & INTERPOLATION LAYER: SUBDIVIDES LINES AND CIRCLES INTO MILLIMETER-SCALE SEGMENTS (0.3MM - 0.5MM) AND CALCULATES INVERSE KINEMATICS FOR EACH SUB-POINT TO GENERATE CONTINUOUS, FLUID TOOLPATHS.
